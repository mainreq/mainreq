¿Cómo administrar MainReq?

Desde el módulo de administración de MainReq se pueden crear y editar Proyectos, Roles (con permisos de edición) y Usuarios.

¿Cómo crear un Proyecto?

  • Ingresar los atributos del nuevo proyecto
    • nombre
    • descripción
    • fecha y semestre de inicio
    • fecha de cierre (tras la cual usuarios no administradores no pueden editar el proyecto)
  • Crear Roles (de no existir) y especificar sus permisos de edición
    • nombre
    • permisos de edición
      • Editor Documentos
      • Editor Hitos
      • Editor Tipos de Usuario
      • Editor Requisitos de Usuario
      • Editor Requisitos de Software
      • Editor Módulos
      • Editor Casos de Prueba
      • Asignar y Revisar Tareas
    • * Generalmente los Roles se crean sólo una vez, pero pueden ser modificados y se pueden crear nuevos Roles de ser necesario
    • * No están asociados a un proyecto en particular. El Rol del usuario determina sus permisos de edición sobre todos sus proyectos
  • Crear Usuarios
    • especificar atributos
      • email (necesario para notificar credenciales)
      • contraseña provisional (será cambiada luego)
      • asociar Roles
      • asociar Proyecto
    • ¿El usuario es administrador?
      • Para hacer que el usuario sea administrador todas las opciones bajo la categoría “Permissions” deben estar activadas (Active, Staff status, Superuser status)
  • Notificar a los nuevos usuarios sus credenciales de acceso
    • acceder a la lista de proyectos y seguir el link "Users..." desde el proyecto creado
    • seleccionar nuevos usuarios (checkboxes)
    • enviar email con credenciales de acceso (los usuarios DEBEN tener un email válido), seleccionando "Send Email Notification" y presionando "Go"
      • esta acción es útil en caso de olvido de contraseña, pues genera una nueva contraseña
  • Acceder al Proyecto
    • verificar que nuestro usuario tenga el proyecto asociado
    • acceder mediante link "My Projects ..."
